# 💳 씬파일러 대안신용평가 모델 개발

신용이력이 부족한 청년(씬파일러)을 대상으로, 전통적인 금융이력 대신 **비금융 대안정보**를 활용하여
신용위험을 평가하고 그 **판정 근거까지 설명**할 수 있는 모델을 개발하는 프로젝트입니다.

---

## 📅 프로젝트 개요

- **소속**: 멀티캠퍼스 데이터 분석가 11기
- **팀원**: 김태헌, 서목원, 염서윤, 천태영
- **주제**: 신용이력 부족 청년(씬파일러)을 위한 대안정보 기반 신용평가 모델
- **대상**: 신용이력이 부족한 청년에게 자금을 공급하는 금융기관 (은행 · 청년 대출 보증기관)

---

## 💡 기획 의도

### 1. 문제의식 — 신용 사각지대에 놓인 씬파일러
기존 신용평가는 카드·대출 등 **과거 금융거래 이력**을 중심으로 이루어집니다.
그 결과 이런 이력이 거의 없는 사회초년생(씬파일러)은 실제로는 성실히 돈을 갚을 사람이어도,
**단지 판단할 정보가 없다는 이유로** 대출이 거절되거나 높은 금리를 부담하게 됩니다.

### 2. 해결 방향 — 대안정보로 신용을 다시 읽다
공공요금 납부, 생활 안정성(주소·직장·연락처 변경 빈도), 소득·소비 패턴 등
**금융이력 외의 대안정보**를 활용해 씬파일러의 상환능력을 더 정확하게 평가합니다.

### 3. 차별점 — 설명 가능한 평가
단순히 점수만 내는 것이 아니라, **SHAP** 기법으로
"어떤 요인이 점수를 올리고 내렸는지"를 제시하여
금융기관이 신뢰하고 활용할 수 있는 **설명가능형 모델**을 지향합니다.

---

## 🎯 핵심 목표

- 신용거래 이력이 없는 씬파일러를 데이터 내에서 신뢰성 있게 **식별**
- 기존 신용점수보다 위험 **변별력이 향상된 모델** 개발
- **SHAP** 기반으로 판정 근거를 설명
- 결과를 **인터랙티브 대시보드**로 시각화하여 누구나 이해할 수 있도록 전달

## 🛠️ 분석 프로세스

**Step 1. 데이터 수집 및 구조 파악** — 변수 사전 작성, 씬파일러 정의 로직 합의

**Step 2. 씬파일러 식별** — 카드·대출 보유건수(CB 원본 필드) 전부 0 기준, Track A/B 공통 적용
(2026-07-09 기준 재정의, 상세: `docs/씬파일러_정의_재검토.md`)

**Step 3. 대안변수 전처리 및 분석** — 공공요금 납부·생활 안정성·소득·소비 변수 정제, 변수별 연체율·변별력 시각화

**Step 4. 모델링** — 기존 신용점수 단독 모델 vs 대안정보 결합 모델 비교

**Step 5. 설명가능성 확보** — SHAP으로 변수별 기여도 도출

**Step 6. 대시보드 시각화** — 모델 성능·개인별 판정 근거·변수 분석을 웹으로 전달

## 📁 폴더 구조

thinfiler-credit-scoring/
├── notebooks/
│   ├── track-a/       # Track A(개인CB정보, Sheet 09) 담당자 작업 노트북
│   └── track-b/       # Track B(통신·소비 결합정보, Sheet 11) 담당자 작업 노트북
├── docs/               # 기획서, 변수사전, 씬파일러 정의 등 팀 공유 문서
├── src/                # 트랙 공통으로 재사용하는 함수 (전처리, 씬파일러 정의 로직 등)
└── outputs/
└── figures/        # 발표·보고서용 시각화 이미지

- **notebooks/track-a, track-b**: 데이터 양이 많아 Sheet09 / Sheet11로 담당을 나눴습니다. 각자 맡은 트랙 폴더에 데이터 탐색부터 전처리·변수 검증까지의 작업 노트북을 저장합니다. 모델링 단계는 역할 확정 후 별도 폴더(`notebooks/modeling` 등)로 분리 예정입니다.
- **docs**: 기획서, 변수사전, 씬파일러 정의 관련 문서를 모아둡니다. 원본 데이터는 올리지 않습니다.
- **src**: 두 트랙에서 공통으로 쓰는 함수를 모아둡니다. 처음엔 비어있고, 노트북 작업 중 반복되는 코드가 생기면 그때그때 채웁니다.
- **outputs/figures**: 대시보드·발표자료에 쓸 시각화 이미지를 저장합니다. 원본 데이터나 모델 파일은 올리지 않습니다.
